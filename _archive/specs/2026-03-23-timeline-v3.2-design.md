# Timeline v3.2 — Design Specification

**Date:** 2026-03-23
**Status:** Draft
**Scope:** Canvas height fix, default zero selection, monthly vertical gridlines, mouse drag panning

---

## 1. Canvas Height Fix

### Problem
`CANVAS_HEIGHT = 560` with `PADDING_Y = 30` places month labels inside the stage area. Currently `scoreToY` uses the full canvas height minus symmetrical padding for 8 stages, leaving only 18px between the -4 stage line and the month labels. The horizontal scrollbar further obscures the labels. The full +4 to -4 range plus axis labels must be visible without vertical clipping.

### Root cause
The stage grid and axis labels compete for the same vertical space because `STAGE_HEIGHT` is derived from the full canvas height minus symmetrical top/bottom padding. Simply increasing `CANVAS_HEIGHT` scales everything proportionally — the gap between the -4 line and labels stays 18px.

### Change
Break the symmetrical padding model by introducing a dedicated axis label area below the stage grid.

**New constants:**
```
CANVAS_HEIGHT = 620
PADDING_Y = 30              (unchanged — top padding only)
AXIS_LABEL_HEIGHT = 40      (new — dedicated bottom area for month/year labels)
STAGE_HEIGHT = (CANVAS_HEIGHT - PADDING_Y - AXIS_LABEL_HEIGHT) / 8
           = (620 - 30 - 40) / 8
           = 68.75px per stage
GROUND_Y = PADDING_Y + 4 * STAGE_HEIGHT
         = 30 + 275
         = 305
```

**`scoreToY` remains the same formula:** `PADDING_Y + (4 - score) * STAGE_HEIGHT`
- Score +4 (Orbit): y = 30
- Score 0 (Watch/Ground): y = 305
- Score -4 (Buried): y = 580

**Month labels** render in the dedicated bottom area:
- Month abbreviations at y = `CANVAS_HEIGHT - AXIS_LABEL_HEIGHT + 18` = 598
- January year text at y = `CANVAS_HEIGHT - AXIS_LABEL_HEIGHT + 30` = 610

**Background zones** update:
- Above ground: y = `PADDING_Y`, height = `GROUND_Y - PADDING_Y` = 275
- Below ground: y = `GROUND_Y`, height = `scoreToY(-4) - GROUND_Y` = 275

**Container height:** 620 + 44 (toolbar) = 664px

**Vertical gridlines** (section 3) run from `PADDING_Y` to `scoreToY(-4)` = 580, stopping above the label area.

### Files changed
- `TimelineCanvas.tsx` — update `CANVAS_HEIGHT`, add `AXIS_LABEL_HEIGHT`, update `STAGE_HEIGHT` and `GROUND_Y` derivation, update month label y-positions, update background zone heights

## 2. Default Zero Selection

### Problem
`getDefaultSelection` picks up to 3 objectives sorted by momentum score. If an objective gets removed, goes stale, or data changes, the auto-selection can break or feel arbitrary.

### Change
- `getDefaultSelection` returns `new Set<string>()` (empty set)
- The existing empty state overlay from v3.1 handles this: italic serif prompt "Select an objective to view its trajectory" centered on the canvas
- Grid, ground line, stage labels, month labels, gridlines, and today marker all render — only trajectories are absent
- User picks what they want to see

### Files changed
- `TimelineCanvas.tsx` — simplify `getDefaultSelection` to return empty set

## 3. Monthly Vertical Gridlines with January Emphasis

### Problem
The timeline lacks vertical rhythm. Without gridlines, it is difficult to visually align nodes with their month positions, especially when trajectories span multiple years.

### Change
Add vertical lines at each month column, rendered in SVG alongside the existing horizontal stage lines.

- **Regular months:** 0.5px stroke, `var(--border)` colour, opacity 0.15
- **January:** 0.5px stroke, `var(--border)` colour, opacity 0.3
- Lines run from `PADDING_Y` (y=30) to `scoreToY(-4)` (y=580) — the full stage area, stopping above the axis label area
- Data source: existing `monthLabels` memo — a `<line>` per entry, no new data structure needed
- Rendering order: gridlines render before paths and nodes (behind everything), alongside horizontal stage lines

### Files changed
- `TimelineCanvas.tsx` — add vertical gridline rendering block within SVG

## 4. Mouse Drag Panning

### Problem
Navigating the timeline requires the scrollbar or keyboard arrow keys. Users expect to click and drag a timeline-style canvas to pan it.

### Change
Add click-and-drag horizontal panning to the scrollable canvas area.

#### Cursor behaviour
- `cursor: grab` when hovering over the canvas scroll container
- `cursor: grabbing` while actively dragging

#### Drag mechanics
- `mousedown`: record start X position and current `scrollLeft`, switch cursor to `grabbing`
- `mousemove` (while dragging): only activate drag after a 5px dead zone (prevents accidental drag on click); then update `scrollLeft` by the delta from start position
- `mouseup` / `mouseleave`: stop dragging, revert cursor to `grab`
- Drag only affects horizontal scroll — no vertical movement
- Apply `user-select: none` during drag to prevent text selection
- Native scroll (scroll wheel, trackpad gestures) must remain functional alongside drag panning

#### Implementation
- Three `useRef` values (not `useState` — avoids re-renders during drag): `isDragging`, `startX`, `startScrollLeft`
- Handlers on the existing `scrollRef` div (the `overflow-x-auto` container)
- `onMouseDown`, `onMouseMove`, `onMouseUp`, `onMouseLeave`
- Existing keyboard navigation (left/right arrow keys) remains unchanged

#### Interaction with existing behaviours
- Node `onMouseEnter`/`onMouseLeave` for tooltips still works normally — drag only activates on mousedown + movement
- A click without movement does not trigger drag, so clicking signal/origin nodes to navigate to Evidence still works

### Alternative considered and rejected
Re-introducing `@panzoom/panzoom` — overkill for horizontal-only panning, and it was intentionally removed in v3.

### Files changed
- `TimelineCanvas.tsx` — drag handlers + cursor styles on scroll container

## Non-goals

These items are explicitly out of scope:
- Vertical drag/panning
- Viewport-responsive dynamic canvas height
- Changes to the legend sidebar layout or styling
- Changes to tooltips, node types, or path rendering
- Mobile/touch drag support

## Test Impact

### Tests to update
- `TimelineCanvas.test.tsx` — update default selection assertion from `"3 of 3 selected"` to `"0 of 3 selected"` (or equivalent count reflecting zero selection)

### New tests needed
- Default zero selection: canvas renders empty state overlay on mount
- Monthly gridlines: SVG contains vertical line elements with correct opacity
- January gridlines: January lines have stronger opacity (0.3) than regular months (0.15)
- Drag panning: mousedown + mousemove updates scrollLeft; mouseup stops dragging
- Drag vs click: mousedown + mouseup without movement does not affect scroll position

### Visual acceptance criteria
- Month labels and year text are fully visible below the -4 stage line with no overlap or clipping
- Gridlines are barely perceptible — they must not compete visually with trajectory paths
