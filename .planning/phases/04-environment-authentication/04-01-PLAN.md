## Plan 04-01: Backend & Frontend Environment Configuration

**Wave:** 1 (parallel-ready)
**Autonomous:** Yes
**Duration:** ~20 minutes

### Objective

Configure local `.env.local` files for backend agent and Next.js frontend. Verify that:
1. Backend agent (`backend/agent.py`) loads without `MissingEnvironmentVariable` errors
2. Frontend dev server starts without Supabase configuration warnings

This plan satisfies **ENV-01** (all backend env vars) and **ENV-02** (all frontend env vars).

### Context

**What exists:**
- `backend/agent.py` — expects ANTHROPIC_API_KEY, SUPABASE_URL, SUPABASE_SERVICE_KEY, FIRECRAWL_API_KEY
- `frontend/` — Next.js app expects NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY
- No `.env.local` files currently (templates exist in `.env.example`)

**What's needed:**
- Get Supabase credentials from dashboard (URL, anon key, service key)
- Get Anthropic API key from account
- Get Firecrawl API key from account
- Store values in `.env.local` files (git-ignored)
- Verify both systems load without errors

### Tasks

#### Task 1: Create backend `.env.local` from template

**Files:**
- `backend/.env.local` (create)
- `backend/.env.example` (read for reference)

**Action:**
Copy `backend/.env.example` to `backend/.env.local`. Fill in the following values from your accounts:

1. `ANTHROPIC_API_KEY` — From [console.anthropic.com](https://console.anthropic.com) → API keys
2. `SUPABASE_URL` — From Supabase Dashboard → Settings → API
3. `SUPABASE_SERVICE_KEY` — From Supabase Dashboard → Settings → API (service_role key, marked "Secret")
4. `FIRECRAWL_API_KEY` — From [Firecrawl app](https://www.firecrawl.dev) → API keys

Do NOT commit `.env.local` to git (should already be gitignored).

**Verify:**
```bash
cd backend
grep -E "ANTHROPIC_API_KEY|SUPABASE_URL|SUPABASE_SERVICE_KEY|FIRECRAWL_API_KEY" .env.local | wc -l
# Should output: 4
```

**Done:** `backend/.env.local` exists, contains all 4 keys, file is git-ignored.

---

#### Task 2: Create frontend `.env.local` from template

**Files:**
- `frontend/.env.local` (create)
- `frontend/.env.example` (read for reference)

**Action:**
Copy `frontend/.env.example` to `frontend/.env.local`. Fill in:

1. `NEXT_PUBLIC_SUPABASE_URL` — Same value as backend's SUPABASE_URL
2. `NEXT_PUBLIC_SUPABASE_ANON_KEY` — From Supabase Dashboard → Settings → API (anon public key)

These are intentionally `NEXT_PUBLIC_` because they're safe for public consumption (the anon key is designed to be public; it has limited RLS-enforced permissions).

**Verify:**
```bash
cd frontend
grep -E "NEXT_PUBLIC_SUPABASE_URL|NEXT_PUBLIC_SUPABASE_ANON_KEY" .env.local | wc -l
# Should output: 2
```

**Done:** `frontend/.env.local` exists, contains both keys, file is git-ignored.

---

#### Task 3: Verify backend agent loads without env errors

**Files:** (read-only)
- `backend/agent.py`
- `backend/.env.local`

**Action:**
Run the agent in "check mode" to verify it can import and load environment variables:

```bash
cd backend
python -c "from dotenv import load_dotenv; load_dotenv(); import os; assert os.getenv('ANTHROPIC_API_KEY'), 'ANTHROPIC_API_KEY missing'; assert os.getenv('SUPABASE_URL'), 'SUPABASE_URL missing'; assert os.getenv('SUPABASE_SERVICE_KEY'), 'SUPABASE_SERVICE_KEY missing'; assert os.getenv('FIRECRAWL_API_KEY'), 'FIRECRAWL_API_KEY missing'; print('✓ All backend env vars present')"
```

If any key is missing, the script will fail with an assertion error. Fix by adding the missing key to `.env.local`.

**Verify:**
```bash
cd backend
python -c "from dotenv import load_dotenv; load_dotenv(); import os; [assert os.getenv(k) for k in ['ANTHROPIC_API_KEY', 'SUPABASE_URL', 'SUPABASE_SERVICE_KEY', 'FIRECRAWL_API_KEY']]" && echo "✓ All 4 backend env vars loaded"
```

**Done:** Command succeeds and prints "✓ All 4 backend env vars loaded".

---

#### Task 4: Verify Next.js frontend loads without Supabase warnings

**Files:** (read-only)
- `frontend/.env.local`
- `frontend/next.config.ts`
- `frontend/src/lib/supabase.ts`

**Action:**
Build the Next.js app in dev mode and verify Supabase client initializes:

```bash
cd frontend
npm run dev --port 3001 &
DEV_PID=$!
sleep 5
curl -s http://localhost:3001 | grep -q "<!DOCTYPE html" && echo "✓ Frontend loaded" || echo "✗ Frontend failed"
kill $DEV_PID 2>/dev/null || true
```

Expected: Frontend responds with HTML (dev server started). Check terminal output for Supabase warnings. Should see NO errors like "NEXT_PUBLIC_SUPABASE_URL is undefined" or similar.

**Verify:**
```bash
cd frontend
npm run build 2>&1 | grep -i "supabase" | grep -i "undefined" && echo "✗ Supabase config issue" || echo "✓ Supabase config loaded"
```

If build shows any "undefined" warnings for Supabase vars, check that `frontend/.env.local` has both keys and they're not empty.

**Done:** Frontend builds without Supabase config errors.
