       ```
       gh workflow run agent-run.yml --repo robopunk/drift-magazine
       ```

    2. Wait 10 seconds, then get the run ID:
       ```
       gh run list --repo robopunk/drift-magazine --workflow=agent-run.yml --limit=1 --json databaseId,status,conclusion
       ```

    3. Watch the run until completion (poll every 30 seconds, timeout after 15 minutes):
       ```
       gh run watch <RUN_ID> --repo robopunk/drift-magazine --exit-status
       ```

       ```
       gh run view <RUN_ID> --repo robopunk/drift-magazine --json status,conclusion
       ```
       until status is "completed".

    4. Once completed, capture the full log output:
       ```
       gh run view <RUN_ID> --repo robopunk/drift-magazine --log
       ```

    5. Verify the run conclusion is "success" (exit code 0). If it fails:
       - Read the log output to identify the error
       - Document the error in the SUMMARY
       - Do NOT retry automatically — report the failure for diagnosis

    IMPORTANT: The run may take 5-10 minutes due to Claude API calls with web search.
    The Anthropic org rate limit is 30k TPM — if this executor session is also consuming tokens,
    the agent run may hit rate limits. If the run fails with a rate limit error, wait 5 minutes
    and trigger a second attempt.
  </action>
  <verify>
    <automated>gh run view <RUN_ID> --repo robopunk/drift-magazine --json status,conclusion --jq '.conclusion' | grep -q "success"</automated>
  </verify>

    - Run log does NOT contain "AuthenticationError" or "401" or "403"
