## Stagger Contract (Proximity-Bucket)

Source: CONTEXT D-C1, D-C2, D-C3.

### Bucket computation (in TimelineCanvas.tsx)

```
bucketKey = Math.round(x / 5)
```

Group all nodes by `bucketKey`. Within each bucket, assign stagger rank 0, 1, 2, 3 in render order (objective order within the bucket). Rank maps to tick height as follows:

| Rank | Origin tick height | Signal/latest tick height |
|------|--------------------|---------------------------|
| 0 | 20px | 20px |
| 1 | 32px | 32px |
| 2 | 44px | 44px |
| 3 | 56px | 56px |

- Origin tick heights and signal/latest tick heights are bucketed **independently** — each set maintains its own rank sequence per bucket.
- `cadence` and `fiscal-year-end` nodes: exempt from bucketing, no tick.
- Pass computed tick height as `tickHeight` prop to `TimelineNode` (replaces `stackIndex` for tick computation).
- The `stackIndex` prop may be removed or repurposed after stagger logic moves to bucket model.

## Animation Contract

Source: CONTEXT D-D1, D-D2, D-D3.

### `@keyframes node-pulse`

Define in `globals.css`:

```css
@keyframes node-pulse {
  0%   { transform: scale(1.0); opacity: 0.9; }
  50%  { transform: scale(1.08); opacity: 1.0; }
  100% { transform: scale(1.0); opacity: 0.9; }
}
```

### Application rules

| Property | Value |
|----------|-------|
| Target element | Outer halo `<circle>` of each `latest` node |
| Apply via | CSS class `node-pulse` on the outer halo `<circle>`, OR `animateTransform` SVG element — choose whichever avoids transform-origin issues in the SVG coordinate system |
| Duration | 2.5s |
| Timing function | ease-in-out |
| Iteration | infinite |
| Transform origin | centre of the circle (`cx`, `cy`) — must set `transform-box: fill-box; transform-origin: center` if using CSS animation on SVG element |
| Constraint | NO Framer Motion, NO GSAP |

If `transform-box: fill-box` is not reliably supported in the target SVG rendering context, fall back to SVG `<animateTransform attributeName="transform" type="scale">` with `additive="sum"` so the existing translate is preserved.

## Spacing Scale

This phase operates in SVG coordinate space, not CSS layout. The 8-point scale applies to SVG positional values:

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Minimum gap between label text and tick end |
| sm | 8px | Tick height increment step (20 → 28 base; actual steps: 20, 32, 44, 56) |
| md | 16px | Minimum tick height at rank 0 for origin nodes |
| lg | 24px | — |
| xl | 32px | Tick height at rank 1 |

Exceptions: SVG node radii (r=2, r=2.5, r=4, r=5, r=6, r=8, r=9, r=10) are not on the 8-point scale — these are visual proportions, not layout spacing.

## Typography

All text in this phase is SVG `<text>` rendered in IBM Plex Mono. No CSS layout typography is introduced.

| Role | Size | Weight | Font | Usage |
|------|------|--------|------|-------|
| Exit label | 9px | bold (700) | IBM Plex Mono | terminal node exit label (PROVED / SILENT DROP) |
| Stage label | 11px | bold (700) | IBM Plex Mono | latest node only |
| Date label | 8px | regular (400) | IBM Plex Mono | origin node date (Oct 2023) |
| Stale indicator | 8px | bold (700) | IBM Plex Mono | `!` in stale node |
| Emoji | 18–20px | n/a | system emoji | signal/latest stage emoji only |

Line height: not applicable (SVG `<text>` uses `dy` for vertical positioning, not line-height).

WCAG 2.2 AA: Labels using `var(--foreground)` (#374151 light / #d1d5db dark) against the canvas surface (`var(--background)`) must achieve ≥4.5:1 contrast. Verify both modes:
- Light: #374151 on #f0f8ff → ≥4.5:1 (passes)
- Dark: #d1d5db on #0f172a → ≥4.5:1 (passes)

Labels using `var(--muted-foreground)` (#6b7280 in both modes) against canvas:
- Light: #6b7280 on #f0f8ff → must verify — if contrast falls below 4.5:1, escalate to `var(--foreground)` for date labels
- Dark: #6b7280 on #0f172a → must verify — same escalation rule applies

## SVG Symbol Definitions

Defined once in `TimelineCanvas.tsx` inside the `<svg>` root element:

```xml
<defs>
  <symbol id="icon-proved" viewBox="0 0 12 12">
    <!-- Upward chevron: two strokes meeting at apex -->
    <!-- stroke="currentColor" so it inherits from <use> fill context -->
    <polyline
      points="2,8 6,4 10,8"
      fill="none"
      stroke="currentColor"
      strokeWidth="1.8"
      strokeLinecap="round"
      strokeLinejoin="round"
    />
  </symbol>
  <symbol id="icon-buried" viewBox="0 0 12 12">
    <!-- Downward-pointing filled triangle -->
    <polygon
      points="6,9 2,3 10,3"
      fill="currentColor"
      stroke="none"
    />
  </symbol>
</defs>
```

Usage in `terminal-proved` node:
```xml
<use
  href="#icon-proved"
  x={x - 4}
  y={y - 4}
  width={8}
  height={8}
  color="var(--primary-foreground)"
/>
```

`color` (not `fill`) on `<use>` drives `currentColor` inheritance into the symbol paths.
