1. Open `backend/schema.sql` and append a new V5 migration block AFTER the existing V5 migration comment block (which ends around line 533 with the accountability_tier migration). Add the following block:

```sql
-- ── V5.1 MIGRATION: Firecrawl Source Content ──────────────────────────────
-- Run once against existing databases.
-- Stores the Firecrawl markdown snapshot for audit trail and human verification.
-- Safe for Supabase Postgres 15: nullable column addition is zero downtime.

ALTER TABLE signals ADD COLUMN IF NOT EXISTS source_content text;
```

This adds a nullable `source_content` text column to the `signals` table. The column stores the Firecrawl markdown excerpt that produced each signal, satisfying FR3 (audit trail). Using `IF NOT EXISTS` makes it idempotent.
