  <name>Task 2: Fix prompt injection with XML document boundaries</name>
  <files>backend/agent.py</files>
  <action>
In `prefetch_company_docs` (around line 169-176), replace the divider-based document concatenation with XML-tagged boundaries per Anthropic best practices.

Replace the return block (lines 169-176):
```python
    divider = "=" * 60
    return (
        f"\n\nPRE-FETCHED DOCUMENTS (full text extracted before your search):\n"
        f"{divider}\n"
        + "\n\n---\n\n".join(sections)
        + f"\n{divider}\n"
        "Use these as primary sources. Then use web_search for any additional recent disclosures.\n"
    )
```

With:
```python
    doc_blocks = "\n".join(
        f"<document source='{url}'>\n{content}\n</document>"
        for url, content in _last_prefetch_contents.items()
    )
    return (
        "\n\nPRE-FETCHED DOCUMENTS (full text extracted before your search):\n"
        f"<documents>\n{doc_blocks}\n</documents>\n"
        "Use these as primary sources. Then use web_search for any additional recent disclosures.\n"
    )
```

Key details:
- Each document wrapped in `<document source='...'>` tag with its source URL
- All documents wrapped in outer `<documents>` container
- Uses `_last_prefetch_contents` dict (already populated in the loop above) to get URL-content pairs, avoiding the intermediate `sections` list format (which prepended `[Source: URL]` as plain text)
- The `sections` list and its loop can remain as-is since `_last_prefetch_contents` is populated as a side effect of the same loop
